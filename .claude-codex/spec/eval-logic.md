# 評価ロジック仕様

> 最終更新: 2026-05-07

---

## 評価の概要

Claudeが投稿を読み、17軸候補から5軸を選んでスコアリングする。
評価結果は `data/evals/{id}.json` に書き込まれ、UIでレーダーチャートとして表示される。

---

## 軸の選定ルール

### 17軸候補

| key | label | 説明 |
|---|---|---|
| `empathy` | 共感性 | 読者が「わかる」と思えるか |
| `originality` | 独自性 | 他にない視点・表現か |
| `specificity` | 具体性 | 具体的な事例・数値・場面があるか |
| `introspection` | 内省度 | 自分自身への深い問いかけがあるか |
| `expandability` | 発展可能性 | 読後にアイデアが広がるか |
| `emotion` | 感情強度 | 感情が前面に出ているか |
| `criticality` | 批評性 | 物事を批判的・多角的に見ているか |
| `humor` | ユーモア | 笑いや軽さがあるか |
| `sociality` | 社会接続性 | 社会・他者・世界との繋がりがあるか |
| `actionability` | 行動性 | 具体的な行動・習慣設計につながるか |
| `expertise` | 専門性 | 専門知識・技術的な深さがあるか |
| `clarity` | 明瞭性 | 読みやすく、言いたいことが明確か |
| `curiosity` | 好奇心 | 「なぜ？」「どうなる？」という探求心があるか |
| `depth` | 思考深度 | 表面でなく深く考えているか |
| `vulnerability` | 素直さ | 弱さ・迷い・不安を正直に書いているか |
| `timeliness` | 時事性 | 今の時代・状況と関連しているか |
| `creativity` | 創造性 | 新しい組み合わせ・発明的な発想があるか |

### 5軸の選び方

1. 投稿を読んで「この投稿の核心はどの軸で説明できるか」を考える
2. 最も強く現れている軸を優先して選ぶ
3. **5軸は固定でなく投稿ごとに変わる**（毎回17軸から選び直す）
4. 似た軸を複数選ばない（例: `depth` と `introspection` が両方強い場合は、より当てはまる1軸だけ選ぶ）
5. `humor` や `vulnerability` のような情動的な軸は、明確に現れているときだけ選ぶ

---

## スコアリングルール

- スコアは **0〜100** の整数
- 平均点は設けない。際立っているから選ばれた軸なので、低スコアになりすぎないようにする
- 目安：

| スコア | 意味 |
|---|---|
| 90〜100 | 投稿の中核をなしている。この軸が主題と言える |
| 70〜89 | 強く現れている |
| 50〜69 | 存在感はあるが部分的 |
| 30〜49 | かろうじて読み取れる程度 |
| 0〜29 | ほとんど見当たらない（この軸を選んだなら再考する） |

- **選んだ5軸のスコアが30以下になるなら、別の軸に変える**

---

## 評価コメントのルール

- 投稿全体への1〜3文のコメント
- もちさん（著者）に語りかける形式
- 何が際立っているか・なぜその軸を選んだかの理由を含める
- 批評的になりすぎず、視点を添えるトーン
- 例：「環境設計で行動を誘導するという視点が鮮明です。〜」

---

## 評価済み判定

- `data/evals/{id}.json` が存在し、`evaluation` キーが null でなければ評価済み
- `get_unevaluated.php` は evals ファイルが存在しないか `evaluation === null` の投稿IDを返す

---

## 評価モードの手順（Claude操作）

1. `php api/cli/get_unevaluated.php` → 未評価投稿IDリストを取得
2. 各IDについて `php api/cli/get_post.php --id=xxx` → 投稿内容を取得
3. 上記ルールに従い5軸・スコア・コメントを生成
4. `php api/cli/write_eval.php --id=xxx --json='{"evaluation":{...},"replies":[]}'` → 書き込み

### write_eval.php に渡すJSONの形式

```json
{
  "evaluation": {
    "comment": "環境設計で行動を誘導するという視点が鮮明です。",
    "axes": [
      { "key": "actionability", "label": "行動性",  "score": 88 },
      { "key": "creativity",    "label": "創造性",  "score": 76 },
      { "key": "specificity",   "label": "具体性",  "score": 72 },
      { "key": "introspection", "label": "内省度",  "score": 65 },
      { "key": "humor",         "label": "ユーモア","score": 58 }
    ],
    "generated_at": "2026-05-06T12:00:00"
  },
  "replies": []
}
```

`replies` は既存のリプライを保持するために渡す（新規評価時は空配列）。

## 迷いやすいケースの対応

| 状況 | 対応 |
|---|---|
| 思考が深いが内省でも発展でも説明できる | 投稿の「向き」で判断。自分を掘るなら `introspection`、外に広がるなら `expandability` |
| ユーモアはあるが笑いが主題でない | `humor` を選ばず、実質的に強い軸を選ぶ |
| 専門的な内容だが文章が短い | `expertise` のスコアは高くなりうる。軸選定は内容の質で判断 |
| どの軸も平均的で突出しない | 最も際立つ5軸を選ぶ。5軸全て60前後でも問題ない |

## 評価コメントの文章プロトコル

### 基本構造

評価コメント（`evaluation.comment`）は **1〜3文** で構成する。
長くなる場合も3文を超えない。

```
[投稿の核心を一言で](文1) + [なぜそう読めるか・具体的な根拠](文2) + [もちさんへの視点追加/問いかけ](文3、任意)
```

### 文の役割と書き方

| 文 | 役割 | 書き方の指針 |
|---|---|---|
| 文1（必須）| 投稿の核心を捉える | 選んだ上位軸を1〜2語使って「〜という視点が鮮明です」「〜の強度が際立っています」などで開始 |
| 文2（必須）| 根拠を示す | 「〜という記述に」「〜の問いかけから」など本文の具体的な部分を引きながら理由づけ |
| 文3（任意）| もちさん視点の一押し | 「この思考をさらに広げると〜」「〜と照らすと面白い」など、発展や問いかけを添える |

### トーン

- **著者（もちさん）に語りかける形式**。「あなたの投稿は〜」より「〜という視点が」のほうが自然
- **批評的になりすぎない**。分析家としての視点を添えるが、評価・採点の印象を与えない
- **絵文字は使わない**。テキストのみ
- 「〜ですね」「〜ました」などの断定的結語より、「〜です」「〜います」で締める

### NG例と修正例

| NG | 修正 |
|---|---|
| 「この投稿は非常に素晴らしいです！」 | → 感情的な賞賛はしない |
| 「内省度が高いと思います。そして独自性もあります。」 | → 軸を列挙するだけでなく、なぜかを書く |
| 「改善点としては〜」 | → 改善提案ではなく視点の提供にとどめる |
| 「もちさんの深い洞察が感じられます」 | → 根拠なき称賛。具体的な本文への言及を添える |

### 例文

```
環境を設計することで行動そのものを変えるという視点が鮮明です。
「意志力ではなく仕組みで動く」という前提を本文が一貫して支えており、actionabilityとclarityが高い水準で共存しています。
この発想を自分の作業フローにどこまで適用できるか、次の問いが自然に浮かぶ構造になっています。
```

## リプライコメントの文章プロトコル

リプライ（`replies[].comments[]`）は指示内容ごとに形式が異なる。
共通ルールと、主要な指示パターン別のプロトコルを以下に定義する。

### 共通ルール

- **label**（コメント種別の見出し）: 指示の要旨を5〜10字で。例: `「批判的視点」` `「ファクトチェック」`
- **body**（本文）: 300〜600文字を目安。短すぎると薄く、長すぎると読まれない
- もちさんに語りかける形式（「あなたは〜」より「この投稿では〜」のほうが自然）
- 断定しすぎない。「〜と読める」「〜という解釈もできる」のトーン
- Markdown は使わない（改行区切りのみ）

### 指示パターン別プロトコル

#### 「3つの視点で語って」系

3件の comments を生成。それぞれ異なる角度を立てる。

| label の例 | 内容 |
|---|---|
| `批判的視点` | 主張への反論・見落としの可能性・前提への疑問 |
| `共感的視点` | どの点に共感できるか・読者にとっての意味 |
| `実用的視点` | これをどう活かすか・次の行動との接続 |

他にも「哲学的視点」「社会的視点」「感情的視点」などに換えてよい。
**3視点は毎回投稿の内容に合わせて選ぶ**（固定しない）。

#### 「ファクトチェックして」系

1件の comment を生成。

- label: `ファクトチェック`
- body: 投稿内の「事実として読める記述」を1〜3点ピックアップし、それぞれに対して「確認できた／不確か／誤り」の所見を述べる
- Claudeが確証を持てない場合は「確認できていない」と正直に記載する

#### 「深掘りして」「発展させて」系

1件の comment を生成。

- label: `深掘り` or `発展案`
- body: 投稿の核心テーマを1段階抽象化または具体化した問い・視点を展開する

#### 「もちさん視点で見て」系

1件の comment を生成。

- label: `もちさん観点`
- body: もちさんの過去の言動・価値観・特性（把握している範囲で）を踏まえて、この投稿がどう位置付けられるかを考察する
