# UIレイアウト仕様

> 最終更新: 2026-05-07
> 役割: 技術リファレンス（フロントエンドを変更するときに読む）

---

## 全体構成

3ペイン構成（左サイドバー・メインエリア・右パネル）。
右パネルは必要なときのみ表示。

```
通常時:
┌──────────┬──────────────────────────┐
│左サイドバー│ メインエリア（3カラム）   │
└──────────┴──────────────────────────┘

右パネル表示時:
┌──────────┬─────────────┬────────────┐
│左サイドバー│ メインエリア │ 右パネル   │
└──────────┴─────────────┴────────────┘
```

---

## 左サイドバー

- 折りたたみ可能（トグルボタンで開閉）
- 内容: アカウント切替・カテゴリフィルター・ナビゲーション
- 折りたたみ時はアイコンのみ表示（幅縮小）

## メインエリア（タイムライングリッド）

- CSS Container Queries でカラム数を自動調整
- 利用可能幅に応じて 1〜3 カラム（最大3列）
- 右パネル開閉でコンテナ幅が変化し、カラム数が自動追従
- スマホ: 縦1カラム（コンテナクエリで自動）
- 参考UI: Misskey / Twitter のカード型フィード

## 右パネル

- **デフォルト: 非表示**
- 開くトリガー:
  - 投稿カードをクリック → 投稿詳細モードで開く
  - 投稿作成ボタンをクリック → 作成フォームモードで開く
- 閉じる: パネル内の閉じるボタン（✕）のみ
- 2つのモード（詳細 / 作成）でパネルを共有

## 投稿カード（グリッド内）

- クリックで右パネルに詳細を表示
- カードに表示する情報は別途 `spec/ui-post-card.md` に定義

## レスポンシブ

| 画面幅 | 左サイドバー | グリッド | 右パネル |
|---|---|---|---|
| デスクトップ | 展開 | 1〜3カラム（自動） | 必要時のみ |
| タブレット | 折りたたみ推奨 | 1〜2カラム（自動） | 必要時のみ |
| スマホ | 非表示 or ドロワー | 1カラム | フルスクリーン |
