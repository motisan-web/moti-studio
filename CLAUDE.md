# 個人SNS（思考ログ基地）

## 目的
もちさん専用の思考ログ・メモ・ツイート元投稿のSNS。
自己分析・リアクション・Claude分析を統合したパーソナルナレッジツール。

## 設計・環境
- PHP 8.2 / XAMPP（ローカル: git15.local）/ Xserver（本番: d00e.motisan.info）
- データストア: JSON（`data/`以下）、SQL不使用
- UI: デスクトップ優先・3ペインダッシュボード（YouTube Studio風）、スマホ時は縦列
- 認証: アカウントごとパスワード、失敗5回/15分でロックアウト

## 制約（常に守る）
- Claude APIは使わない。Claudeがチャットでコマンドを打ちPHPヘルパーを実行する
- `data/` は gitignore 対象（ローカルと本番どちらにも存在するが git 管理外）
- SYNC_KEY などの秘密値はファイルに書かない（環境変数で渡す）

## ドキュメント体系

### 毎チャット必読

| ファイル | 役割 |
|---|---|
| `CLAUDE.md`（このファイル） | プロジェクトルール・制約・ドキュメント地図・todo/issue |
| `.claude-codex/CURRENT.md` | セッション状態・直近完了・次にやること・注意事項 |

この2ファイルを順に読めば作業を始められる状態にする。

---

### モード起動時に読む（操作ガイド）

Claude に操作を依頼したとき、該当ファイルだけ読めば完結する。他のファイルを読む必要はない。

| 起動ワード | 読むファイル |
|---|---|
| 「評価して」 | `.claude-codex/spec/eval-mode.md` |
| 「カテゴリを付けて」 | `.claude-codex/spec/category-logic.md` |
| 「draftsを投稿して」 | `.claude-codex/spec/claude-modes.md`（投稿モードの項） |
| 「リプライして」 | `.claude-codex/spec/claude-modes.md`（リプライモードの項） |
| 「同期して」 | `.claude-codex/spec/sync.md` |

---

### 実装・変更時に参照（技術リファレンス）

機能を追加・変更するときに必要に応じて読む。

| ファイル | 内容 | 使うとき |
|---|---|---|
| `spec/data-schema.md` | 全JSONスキーマ定義 | データ構造を確認・変更するとき |
| `spec/api.md` | APIエンドポイント一覧と仕様 | APIを追加・修正するとき |
| `spec/auth.md` | 認証ロジック（ロックアウト等） | 認証まわりを触るとき |
| `spec/ui-layout.md` | UI構造・3ペインレイアウト | フロントエンドを変更するとき |
| `spec/eval-logic.md` | 評価軸17候補の詳細定義 | 評価ロジックを修正するとき |
| `spec/sync.md` | 同期仕様・マージルール・API設計 | 同期機能を修正するとき |

---

### 作業履歴（必要なときだけ）

| ファイル | 内容 |
|---|---|
| `.claude-codex/change/*.md` | 完了した作業の詳細ログ |

## コミットルール

### 粒度
- **1作業単位 = 1コミット**。spec変更・実装・ドキュメント更新はそれぞれ別コミットにする
- 「ついでに直した」修正も別コミットに分ける
- 複数ファイルをまとめて1コミットにしてよいのは、同一の目的で不可分な変更のみ

### メッセージ形式
```
種別: 内容の要約（日本語）

種別一覧:
  feat   - 新機能の追加
  fix    - バグ修正
  spec   - 仕様ドキュメントの追加・更新
  docs   - CLAUDE.md / CURRENT.md / change/ の更新
  refactor - 動作を変えないリファクタリング
  style  - UIスタイル変更
```

例:
```
feat: 同期CLIスクリプト（tools/sync.php）を追加
spec: sync.md に同期仕様を追記
docs: CURRENT.md を最新状態に更新
```

## ドキュメント管理ルール

### todo のルール
- `[ ]` のみここに記載する
- `[x]` にしたら**その行を削除**する（記録は change/ の変更ログが担う）

### issue のルール
- バグ・不具合・改善点が見つかったら、**まずここに `[ ]` で登録してから対処**する
- 対処完了後は `[x]` にしてから**その行を削除**する

### 変更ログのルール
- 作業の変更内容は `.claude-codex/change/作業内容.md` に記録する
- 記録フォーマット：タイムスタンプ・変更内容・意図を羅列する

### spec 更新ルール
- **ファイルを変更するたびに、対応する spec ドキュメントも必ず更新する**
- 実装前に spec を書き、実装後に差分があれば即座に spec を修正する
- spec ファイル名規則：`spec/機能名.md`（例: `spec/api.md`、`spec/sync.md`）

### セッション引き継ぎのルール
- チャット終了時に `.claude-codex/CURRENT.md` を最新状態に更新する
- 新チャット開始時は **CLAUDE.md → CURRENT.md** の順に読めば文脈が揃う状態を保つ
