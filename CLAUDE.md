# 個人SNS（思考ログ基地）

## 目的
もちさん専用の思考ログ・メモ・ツイート元投稿のSNS。
自己分析・リアクション・Claude分析を統合したパーソナルナレッジツール。

## 設計・環境
- PHP 8.2 / XAMPP（ローカル: git15.local）/ Xserver（本番: xxxx.motisan.info）
- データストア: JSON（`data/`以下）、SQL不使用
- UI: デスクトップ優先・3ペインダッシュボード（YouTube Studio風）、スマホ時は縦列
- 認証: アカウントごとパスワード、失敗5回/15分でロックアウト、既存セッション無効化なし

## 主要機能
- **投稿**: タイトル任意、本文、画像添付可、アカウント選択
  - Web UIまたはClaudeが`data/drafts/`のMDを読んでAPI投稿
  - MD区切り文字: `---`、フロントマターでtitle/account指定
- **カテゴリ**: Claudeが自動付与、候補にない場合は新規作成
  - 判定ロジックは `.claude-codex/spec/category-logic.md` に定義済み
  - 特殊カテゴリ: `もちさん設定`（もちさんのキャラクター・設定）、`もち関連`（日常・エピソード）は基本セットで両方付与
- **アーカイブ**: 投稿ごとに期限設定可（カレンダー選択 or 1週間後ボタン）、期限超過時にアクセス時点でアーカイブへ移動（UI非表示）
- **リアクション**: Slack風カスタム絵文字（slug+画像、横長・GIF可）、回数制限なし
- **リプライ**: 投稿IDをClaudeに送ると評価コメントが届く
- **自己コメント**: もちさんが後から突っ込み・補足を追加
- **ラベル**: 投稿にメタ情報としてフラグを付ける（アクション完了の記録用途）
  - 種別: Misskey投稿済み / Twitter投稿済み / 解決済み / 実装済み / キャンセル / 検証済み
  - 各ラベルにURL・メモを添付可能
  - 投稿詳細パネルから追加・削除
- **分析（非リアルタイム）**: 17軸候補から5軸ピック→5角形グラフ、もちさん視点を踏まえた評価
- **複数アカウント**: moti, moti2... でジャンルや視点を区切る
- **検索**: 全文検索 + ランダム1件取得API

## Claude操作モード

Claude APIは使用しない。Claudeがチャットでコマンドを実行しPHPヘルパー経由で処理する。
各モードの詳細手順は `.claude-codex/spec/claude-modes.md` を参照。

| モード | 起動条件 | 読むべきspec |
|---|---|---|
| カテゴリ付与 | 「カテゴリを付けて」など | `spec/category-logic.md` |
| 評価 | 「評価して」など | `spec/data-schema.md` |
| 投稿（draftsから） | 「draftsを投稿して」など | なし |
| リプライ | 投稿IDと指示を受け取ったとき | なし |

## ドキュメント管理ルール

### ファイルの役割分担

| ファイル | 役割 | 読むタイミング |
|---|---|---|
| `CLAUDE.md` | プロジェクトルール＋アクティブな todo/issue のみ | 毎チャット必読 |
| `.claude-codex/CURRENT.md` | 直近の完了内容・次にやること・注意事項 | 毎チャット必読 |
| `.claude-codex/change/*.md` | 完了した作業の詳細ログ | 必要なときだけ |

### todo のルール
- `[ ]` のみここに記載する
- `[x]` にしたら**その行を削除**する（記録は change/ の変更ログが担う）
- 変更ログファイル名を `（ファイル名.md）` の形で末尾に添えてチェックする

### issue のルール
- バグ・不具合・改善点が見つかったら、**まずここに `[ ]` で登録してから対処**する
- 対処完了後は `[x]` にしてから**その行を削除**する

### 変更ログのルール
- 作業の変更内容は `.claude-codex/change/作業内容.md` に記録する
- 記録フォーマット：タイムスタンプ・変更内容・意図を羅列する
- 記録タイミング：todo/issue を消化したとき・変更に一区切りがついたとき

### ファイル仕様ドキュメントのルール
- 各ファイル（PHP/JS/CSS/APIエンドポイントなど）は、対応する仕様ドキュメントを `.claude-codex/spec/` 以下に持つ
- ファイル名規則：`spec/ファイルパスをスラッシュ区切りでハイフン化.md`（例: `spec/api-posts.md`、`spec/js-app.md`）
- 仕様ドキュメントには「役割・入出力・依存・データ構造・制約」を記載する
- **ファイルを変更するたびに、対応する spec ドキュメントも必ず更新する**
- 実装前に spec を書き、実装後に差分があれば即座に spec を修正する

### セッション引き継ぎのルール
- チャット終了時に `.claude-codex/CURRENT.md` を最新状態に更新する
- 新チャット開始時は **CLAUDE.md → CURRENT.md** の順に読めば文脈が揃う状態を保つ
